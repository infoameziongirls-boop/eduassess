# EDUASSESS - RENDER DEPLOYMENT GUIDE

## Overview
This guide explains how to deploy the EDUASSESS application on Render with:
- Persistent PostgreSQL database
- Automatic backups every 6 hours
- Health checks every 12 hours
- Zero data loss between deployments

## Prerequisites
1. Render account (https://render.com)
2. GitHub repository with this code
3. Database credentials (will be provided by Render)

## Step 1: Create PostgreSQL Database on Render

1. Log in to Render Dashboard
2. Click "+ New +" → "PostgreSQL"
3. Configure:
   - **Name**: `eduassess-db` (or your choice)
   - **Database**: `eduassess`
   - **User**: Autogenerated (save this)
   - **Region**: Choose closest to your location
4. Click "Create Database"
5. **IMPORTANT**: Save the `Internal Database URL` - you'll need this

## Step 2: Deploy Web Service

### 2.1: Create Web Service

1. Click "+ New +" → "Web Service"
2. Connect your GitHub repository
3. Configure:
   - **Name**: `eduassess` (or your choice)
   - **Environment**: Python 3
   - **Build Command**: `bash build.sh`
   - **Start Command**: `gunicorn wsgi:app`
   - **Region**: Same as database
   - **Plan**: Standard (minimum for production)

### 2.2: Set Environment Variables

Go to **Environment** section and add:

```
FLASK_ENV=production
DATABASE_URL=[PostgreSQL Internal URL from Step 1]
DEFAULT_ADMIN_USERNAME=admin
DEFAULT_ADMIN_PASSWORD=YourSecurePassword
SECRET_KEY=your-secret-key-here
```

**IMPORTANT**: Change `DEFAULT_ADMIN_PASSWORD` to a strong password!

### 2.3: Deploy

1. Click "Create Web Service"
2. Wait for deployment to complete (5-10 minutes)
3. Render will show your app URL (e.g., https://eduassess.onrender.com)

## Step 3: Verify Deployment

1. Visit your app URL
2. Log in with:
   - Username: `admin`
   - Password: (whatever you set as `DEFAULT_ADMIN_PASSWORD`)
3. Check the admin panel to verify data persistence

## Step 4: Setup Automated Backups

### Option A: Background Job (Recommended for Production)

1. Go back to Render Dashboard
2. Click "+ New +" → "Background Worker"
3. Connect same GitHub repository
4. Configure:
   - **Name**: `eduassess-backup-worker`
   - **Environment**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python backup_scheduler.py`
   - **Plan**: Standard
5. Set same Environment Variables as Web Service
6. Click "Create Background Worker"

**This will:**
- Run backups every 6 hours automatically
- Run health checks every 12 hours
- Create JSON backups in the `backups/` directory
- Continue even if the web service restarts

### Option B: Cron Job (If Background Worker Not Available)

Use a free cron service like cron-job.org:

1. Visit https://cron-job.org/
2. Create account and add job
3. Set URL: `https://[your-app-url]/api/backup` (we'll create this endpoint)
4. Schedule: Every 6 hours
5. This triggers backup on your web service

## Step 5: Access Backups

Backups are stored in two locations:

### JSON Backups (Local)
- Location: `/app/backups/` on Render
- These persist in your app's storage
- Downloadable via admin panel (if you add download feature)

### Cloud Backup (Recommended for Additional Safety)

Add AWS S3 or Google Cloud Storage integration:

1. Get your cloud storage credentials
2. Add to Environment Variables:
   ```
   AWS_ACCESS_KEY_ID=...
   AWS_SECRET_ACCESS_KEY=...
   AWS_S3_BUCKET=eduassess-backups
   ```
3. Modify `backup_scheduler.py` to upload to cloud

## Database Persistence

### How Data Persists
- All data stored in PostgreSQL (not in app container)
- PostgreSQL automatically backed up by Render
- When you deploy new code, database stays intact
- When app restarts, no data is lost

### Manual Database Backups
To manually create a database backup:

1. Go to your PostgreSQL database on Render
2. Click "Backups" tab
3. Click "Create Backup"
4. Download if needed

## Monitoring

### Check App Logs
1. Go to Web Service in Render Dashboard
2. Click "Logs" tab
3. Look for:
   - ✓ "Application startup complete"
   - ✓ "Database initialization completed"
   - ✓ "Backup job completed successfully"

### Database Health
- Run health check: `python db_health_check.py` (local)
- View in logs: Render logs → search "health check"

### Check Backups
1. SSH into Render (if available): `render shell`
2. Check backups: `ls -lh backups/`
3. Or download via API if endpoint created

## Common Actions

### Reset Admin Password
1. Go to Render Dashboard → Web Service → Shell
2. Run: `python -c "from app import app, bcrypt, db; from models import User; u = User.query.filter_by(username='admin').first(); u.password_hash = bcrypt.generate_password_hash('newpassword').decode('utf-8'); db.session.commit(); print('Password updated')"`

### Clear Backups
1. SSH into Render
2. Run: `rm backups/*`

### Restore from Backup
Ask support or manually run import script with backup JSON files

## Troubleshooting

### Database Connection Failed
- Check `DATABASE_URL` in Environment Variables
- Verify it's the "Internal Database URL" (not External)
- Check database is in same region

### Backups Not Running
- Check Background Worker logs
- Verify `DATABASE_URL` is set correctly
- Ensure PostgreSQL database is running (not stopped)

### No Data Appears After Deployment
- Wait 1-2 minutes for database initialization
- Check app logs for errors
- Verify you created users/students before deploying

## Migration from Local Development

If migrating existing data:

1. Create backup on local: `python backup_all_data.py`
2. Download `backups/students_backup_*.json` and `assessments_backup_*.json`
3. After deploying on Render, create an import endpoint or import manually
4. Test thoroughly before going live

## Important Security Notes

1. **Change Default Password**: Always change `DEFAULT_ADMIN_PASSWORD`
2. **Use HTTPS**: Render provides free SSL certificates
3. **Backup Database**: Render keeps 7-day backups automatically
4. **Monitor Access**: Check admin logs regularly for unauthorized access
5. **Keep Secrets Secret**: Never commit credentials to GitHub
